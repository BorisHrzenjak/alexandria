## Screenshot Guide

Since this is a validation campaign, here are the screenshots you need to capture. Prioritize dark mode for all visuals (it performs better on Twitter/X for tech audiences).

| # | Screenshot | View | Key elements to show |
|---|-----------|------|---------------------|
| 1 | **Main library** | All Prompts (dark mode) | 6-8 prompts with varied categories, star icons on 2-3 favorites, search bar empty, sidebar collapsed or expanded |
| 2 | **Copy action** | All Prompts (dark mode) | One prompt highlighted/hovered, "Copied!" toast visible |
| 3 | **Playground** | Playground (dark mode) | A realistic prompt in the textarea, model dropdown open or a model selected, "Optimize Prompt" button prominent |
| 4 | **Optimized result** | Playground (dark mode) | The overlay showing an optimized prompt with the action buttons (copy, save, download, retry, restore) |
| 5 | **Dashboard** | Dashboard (dark mode) | Stats cards filled with realistic numbers, category bars populated, activity graph with some data |
| 6 | **Add prompt modal** | Modal open (dark mode) | Title filled in, prompt text visible, category dropdown, tags input |
| 7 | **Category filter** | All Prompts filtered (dark mode) | Category dropdown open showing 4-5 categories, list showing filtered results |
| 8 | **Settings** | Settings modal (dark mode) | Dark mode toggle on, import/export buttons, OpenRouter API key field |

**Tip:** Populate the extension with 15-20 realistic prompts across 4-5 categories before taking screenshots. Use categories like "Coding," "Writing," "Image Gen," "System Prompts," "Data Analysis." This makes the app feel real and lived-in.

## Campaign Notes

**Budget allocation suggestion:** Put 60% of budget on Ad Set 3 (developer/vibecoder angle) and Ad Set 4, Tweet 3 (build-in-public). These formats tend to get the highest engagement from the target audience. Use Ad Set 1 for broader reach.

**Hashtags to test (pick 2-3 per tweet):**
`#AI` `#PromptEngineering` `#ChromeExtension` `#BuildInPublic` `#IndieHacker` `#Vibecoding` `#DevTools` `#AITools` `#OpenSource`

**Validation metrics to watch:**
- Click-through rate (are people curious enough to try it?)
- Retweets/quotes (does the concept resonate enough to share?)
- Replies (what questions or objections come up?)
- Poll results (confirms the pain point exists)
- Install rate from link clicks (real conversion signal)

**Timing:** Post between 9-11 AM EST or 1-3 PM EST on weekdays. The dev/AI community on X is most active during US work hours.
